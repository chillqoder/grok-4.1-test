# Mobile App UI Generation Prompt (Flutter)

## Goal
Design a **modern, premium-quality mobile application UI** built with **Flutter**.

The application must focus ONLY on:
- visual design
- navigation structure
- layout composition
- interaction feel

No backend logic, no real data models, no business rules.

Check assets/data/movies.json

This is a **design-first navigation prototype**.

---

## Core Principles

Create an interface that feels:

- ultra-modern
- premium-level
- minimalist but expressive
- comparable to top-tier global tech products
- clean, confident, and future-oriented

The design must look like it belongs among the best contemporary mobile apps.

Avoid generic templates.

---

## Design Style

### Visual Direction
- Modern mobile design language
- Soft depth and layered surfaces
- Subtle glass / blur / elevation usage
- Smooth rounded corners
- Balanced whitespace
- Elegant typography hierarchy
- Calm but confident color usage

### Inspiration Mood (not brands, just feeling)
- high-end startup apps
- award-winning product design
- modern SaaS mobile interfaces
- futuristic but usable UI

---

## Navigation Requirements

Implement navigation ONLY (no logic).

Include:

- Bottom navigation bar OR floating navigation concept
- 4–5 main sections/screens
- Smooth animated transitions
- Gesture-friendly navigation
- Clear active states
- Micro-interactions between screens

Example structure (can reinterpret freely):
- Home
- Explore
- Activity
- Messages
- Profile

Navigation must feel intuitive and premium.

---

## Screens (Structure Only)

Each screen should contain placeholder UI elements:

- cards
- lists
- headers
- buttons
- avatars
- media blocks
- empty states
- loading skeletons

Use mock content only (fake text/images).

No real functionality required.

---

## UI Components

Design a reusable component system:

- modern buttons (primary / secondary / ghost)
- cards with depth
- modal sheets
- floating action buttons
- segmented controls
- tabs
- search field
- chips / tags
- modern input fields

All components must look consistent and production-ready.

---

## Motion & Interaction

Add premium interaction feel:

- smooth page transitions
- subtle animations
- micro-interactions
- soft easing curves
- responsive press states

Avoid excessive animation.

Motion should feel intentional and refined.

---

## Technical Constraints

- Platform: **Flutter**
- Mobile-first design
- Responsive for different phone sizes
- Clean widget hierarchy
- Reusable widgets preferred
- Follow modern Flutter UI practices

No backend integration.
No API calls.
No state management logic required.

---

## Output Expectations

Generate:

- Flutter UI structure
- navigation scaffold
- multiple screens
- reusable UI widgets
- modern layout composition

Focus on visual excellence over functionality.

The result should feel like a **flagship-quality mobile app interface** ready for further product development.

---

## Priority Order

1. Visual quality
2. Navigation experience
3. Design consistency
4. Modern interaction feel
5. Code clarity

Functionality is NOT required.
